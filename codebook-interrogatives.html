Interrogatives Codebooks

疑问词密码本

These codebooks teach the first published interrogative — 0.51 (solve) — through three worked examples of increasing depth: a single-unknown equation, a two-unknown system that distinguishes which variable is being asked about, and a unit-aware query that distinguishes 2tp from 2 and 2lp. The 0.5x family is open: future members (0.52, 0.53, …) are reserved. 本组密码本通过三个由浅入深的示例，教授第一个公开的疑问词 0.51（求解）：单未知数方程、二元方程组（区分被询问的变量）、 带单位求解（区分 2tp、2、2lp）。 0.5x 系列开放——0.52、0.53… 为后续成员预留。

Example B — Two-Unknown System

示例 B：二元方程组

Statement: x₁ + x₂ = 3 and x₂ − x₁ = 1. The query asks for one variable at a time, and the answer rows include deliberate wrong answers so the receiver can see the boundaries: wrong value → false, and right value but wrong variable → also false. 陈述：x₁ + x₂ = 3 与 x₂ − x₁ = 1。 每次只问一个变量，应答行刻意包含错误样本，让接收者看清边界： 值错→错误；值对但变量对错（问 x₂ 答 x₁=1）→ 仍然错误。

Example C — Unit-Aware Query

示例 C：带单位求解

Statement: 1 t_p + 1 t_p = x where t_p is Planck time (0.31). The query asks for x, and the candidate answers test three distinct failure modes: correct value with the right unit (true), correct value without a unit (false), and correct value with the wrong unit — Planck length (false). 陈述：1 t_p + 1 t_p = x，其中 t_p 是普朗克时间（0.31）。提问求 x， 候选答案覆盖三种失败模式：值对单位对（正确）、值对但漏单位（错误）、 值对但单位换成普朗克长度（错误）。

The three answer rows form a minimal-pair test of dimensional consistency. Together they teach the receiver that the codebook treats 2 t_p, 2, and 2 l_p as three distinct objects, and that a correct solution must preserve both magnitude and the originating unit. This is the bridge that makes 0.51 useful in physics, not just arithmetic. 三条应答构成一个最小对照组，专门检验"量纲一致性"： 2 t_p、2、2 l_p 被密码本视为三个完全不同的对象， 合格的解必须同时保留数值与原单位。这正是把 0.51 从"算术求解"推进到"物理求解"的桥梁。

Why 0.51 Matters

0.51 的意义

With the four earlier families, the sender could only describe. With 0.51, the sender can pose a problem and let the receiver check the answer key — verified within the same transmission via 0.18 1 / 0.18 0. That single addition turns the codebook into a self-testable system: any civilization that decodes 0.51 correctly can re-derive every example, and any mis-decoding shows up immediately as inconsistent verdicts. The reserved slots 0.52, 0.53, … will extend this from "solve" to other question types as the design grows. 没有 0.51 之前，发送者只能"描述"。加入 0.51 后，发送者能给出问题并把"参考答案"附在同一条传输里， 通过 0.18 1 / 0.18 0 完成自验证。仅此一处增量，整个密码本就升级为可自测系统： 任何能正确解码 0.51 的文明都能复算我们给出的每个例题， 而一旦解码错误，判定行立刻显示出矛盾。 预留的 0.52、0.53… 将把"求解"扩展到其它疑问类型。
"If I had an hour to solve a problem I'd spend 55 minutes thinking about the problem and 5 minutes thinking about solutions."
— Adapted from Albert Einstein

Before 0.51, every codebook entry was a finished sentence. After 0.51, the codebook contains questions — open invitations to think. That asymmetry between asking and answering is, perhaps, the deepest feature of intelligence we share with anyone listening.

引入 0.51 之前，密码本里每一条都是说完的句子；引入之后，密码本里出现了"问题"—— 一种邀请对方思考的开放姿态。提问与应答之间的不对称， 也许是我们与远方的倾听者最深处的共通之处。
